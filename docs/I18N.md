# Frontend Localization

ChainBridge frontend supports localized routing and runtime language switching.

## Supported Locales

- `en` (English, default)
- `es` (Spanish)
- `zh` (Chinese)
- `ja` (Japanese)
- `ar` (Arabic, RTL)

## How It Works

- Locale is prefixed in URLs (for example: `/en/swap`, `/ja/orders`).
- `middleware` persists locale in `cb-locale` and rewrites paths.
- `I18nProvider` resolves locale-specific strings and sets document `lang` + `dir`.
- `Navbar` exposes a language switcher for instant locale changes.

## RTL Support

Arabic uses RTL direction automatically via:

- `document.documentElement.dir = "rtl"`

All other locales use LTR.
